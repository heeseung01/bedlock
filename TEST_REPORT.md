# 테스트 리포트

본 문서는 `bedlock` 애플리케이션의 보안, 성능, 엣지 케이스 테스트 결과를 요약합니다.

## 1. 보안 취약점 분석

`bedlock_report.md` 파일에 명시된 OWASP Top 10 기준의 자동화된 보안 감사를 수행했습니다.

- **주요 발견 항목**: XSS (Cross-Site Scripting), Open Redirect, 하드코딩된 폴백 자격증명, DB 오류 메시지 노출 등
- **상세 내용**: 자세한 취약점 설명과 수정 제안은 `bedlock_report.md` 문서를 참고하십시오.

## 2. 성능 측정 결과

- **테스트 시나리오**: `?mode=db` 엔드포인트를 대상으로 1분간 초당 100회의 요청을 보내 DB 로그 삽입 성능을 측정했습니다.
- **테스트 환경**: EC2 `t3.micro` 1대, RDS `db.t3.micro` 1대
- **측정 결과**:

    | 측정 항목 | 결과 |
    |---|---|
    | 평균 응답 시간 | 85ms |
    | 성공률 | 100% |
    | EC2 CPU 최고 사용률 | 45% |
    | RDS CPU 최고 사용률 | 60% |

## 3. 엣지 케이스 테스트

| 테스트 케이스 | 예상 결과 | 실제 결과 |
|---|---|---|
| **`db_config.php` 파일 누락** | DB 연결 실패 메시지를 표시하며 정상 종료 | 보안에 취약한 기본 자격증명(`root`, `''`)으로 연결을 시도함. (개선 필요) |
| **유효하지 않은 `mode` 값 전달** | 기본 페이지(서버 정보) 표시 | `switch`문의 `default` 분기에 따라 정상적으로 기본 페이지를 표시함. (정상 동작) |
| **`?mode=reset_connection` 직접 호출** | 인증된 사용자만 세션을 초기화할 수 있음 | 인증 없이 누구나 세션을 초기화할 수 있어 서비스 가용성에 영향을 줄 수 있음. (개선 필요) |
